# Seoul Rent Comparison Report

서울권 전월세 후보를 비교하기 위한 공유용 정적 대시보드입니다.

배포 URL: https://lh0156.github.io/seoul-rent-comparison-report/

이 저장소는 분석 원천 데이터를 전부 재가공하는 작업 공간이 아니라, 같이 볼 사람에게 결과물을 안정적인 URL로 공유하기 위한 **배포용 저장소**입니다. 핵심 산출물은 루트의 `index.html` 하나이며, GitHub Pages가 이 파일을 그대로 서비스합니다.

## 현재 구성

| 항목 | 값 |
| --- | --- |
| 문서 제목 | 집 후보 비교 대시보드 |
| 배포 방식 | GitHub Pages, `main` 브랜치 루트 |
| 주 파일 | `index.html` |
| 파일 형태 | CSS와 이미지를 포함한 단일 HTML |
| 후보 수 | 41개 |
| 포함 이미지 | 155개, `data:image/...;base64`로 내장 |
| 외부 링크 | 네이버 부동산, 직방, HUG, 법령정보, 서울시 자료 등 |
| 빌드 도구 | 없음 |
| 서버 코드 | 없음 |
| JavaScript | 없음 |
| 폼/입력 수집 | 없음 |

## 저장소 구조

```text
.
├── index.html   # GitHub Pages가 실제로 보여주는 단일 파일 대시보드
├── README.md    # 보고서 구조, 공유 정책, 업데이트 방법
├── .nojekyll    # GitHub Pages에서 Jekyll 처리를 건너뛰기 위한 표시 파일
└── .gitignore   # OS/에디터 임시 파일 제외
```

`index.html`은 공유 편의를 위해 이미지까지 한 파일에 넣은 산출물입니다. 현재 파일 크기는 약 32.6MB입니다. 대신 카카오톡이나 메신저로 HTML 파일을 직접 던지지 않아도 같은 URL에서 최신본을 볼 수 있습니다.

## 대시보드 읽는 법

대시보드는 크게 네 부분으로 보면 됩니다.

1. **결론**
   - 이번 비교에서 적용한 핵심 조건과 제외 기준을 요약합니다.
   - 보증금, 월세, 면적, 반지하 여부, 출퇴근, 방/욕실 수, 집 컨디션을 함께 봅니다.

2. **순위표**
   - 전체 후보를 한눈에 비교하는 표입니다.
   - 월 부담, 보증금, 면적, 위치, 감점 사유, 방문 우선순위를 빠르게 봅니다.

3. **집별 분석**
   - 각 후보별로 사진 요약, 장점, 리스크, 생활권, 현장 확인 포인트를 정리합니다.
   - 원문 매물 링크를 같이 둬서 실제 노출 상태와 조건 변경 여부를 다시 확인할 수 있습니다.

4. **참고 자료**
   - HUG 보증, 주택임대차보호법, 주소/생활권 확인 자료 등 판단 근거가 되는 외부 자료를 연결합니다.

## 평가 기준

현재 보고서의 기본 판단 기준은 다음과 같습니다.

| 기준 | 적용 방식 |
| --- | --- |
| 보증금 | 1억 5천만원 초과 후보는 강하게 감점 |
| 월세 | 월세 175만원 이상 후보 제외 |
| 면적 | 전용 약 15평 이상 우선 |
| 층/형태 | 반지하/지층 제외 |
| 구조 | 방 최소 2개, 욕실 최소 1개 우선 |
| 출퇴근 | 역삼, 신논현, 판교 접근성 고려 |
| 컨디션 | 사진상 마감, 채광, 노후도, 수리 필요성 고려 |
| 계약 리스크 | 등기부, 선순위 보증금, 보증보험 가능성 확인 전제 |

점수는 최종 결정을 자동으로 내려주는 값이 아니라, 방문 우선순위를 정하기 위한 정렬 기준입니다. 실제 계약 판단은 현장 확인, 등기부등본, 건축물대장, 전입세대열람, 선순위 임차보증금, 보증보험 심사 가능 여부를 확인한 뒤 해야 합니다.

## 외부 링크 정책

보고서에는 원문 확인을 위한 외부 링크가 포함되어 있습니다.

- 네이버 부동산 매물 링크
- 직방 공유 매물 링크
- HUG 전세보증금반환보증 안내
- 국가법령정보센터 주택임대차보호법
- 서울시 안심이 앱 안내
- 주소/생활권 참고 페이지

직방 공유 URL에는 `userNo`, `stamp` 같은 쿼리 파라미터가 포함된 링크가 있습니다. 이 파라미터는 원문 공유 링크로 정상 진입하기 위해 유지합니다. 임의로 제거하면 공유 링크가 깨질 수 있으므로, 매물 링크 정리 시에도 원본 URL을 보존합니다.

## 공개 범위

이 저장소는 공유 편의를 위한 공개 정적 페이지입니다. 아래 항목은 포함하지 않습니다.

- API 키
- 토큰
- 비밀번호
- 서버 코드
- 폼 입력
- 방문자 추적 코드
- 이메일 주소
- 전화번호
- 로컬 파일 경로

다만 매물 주소, 가격, 면적, 사진 요약, 원문 링크 등 보고서 판단에 필요한 정보는 공개 페이지에 노출됩니다. 공개되어도 되는 후보 비교 자료만 올리는 저장소로 사용해야 합니다.

## 업데이트 방법

새 공유용 HTML이 만들어졌을 때는 배포용 파일을 교체하고 커밋하면 됩니다.

```bash
cp ../_보고서/공유용_단일파일.html index.html
git status --short
git add index.html README.md .nojekyll .gitignore
git commit -m "Update rent comparison dashboard"
git push
```

GitHub Pages는 `main` 브랜치 루트의 `index.html`을 서비스합니다. 푸시 후 반영까지 보통 수십 초에서 몇 분 정도 걸릴 수 있습니다.

## 검수 체크리스트

배포 전에는 최소한 아래를 확인합니다.

- `index.html`이 브라우저에서 열린다.
- 상단 순위표 앵커가 집별 분석 섹션으로 이동한다.
- 사진이 깨지지 않는다.
- 네이버/직방 원문 링크가 정상 진입된다.
- 직방 공유 링크의 쿼리 파라미터가 유지되어 있다.
- API 키, 토큰, 전화번호, 이메일, 로컬 경로가 들어가지 않았다.
- 월세/보증금 조건 변경이 있으면 보고서 날짜와 판단 메모를 갱신했다.

## 한계

이 보고서는 매물 후보를 빠르게 비교하기 위한 의사결정 보조 자료입니다.

- 매물 가격, 노출 상태, 입주 가능일은 수시로 바뀔 수 있습니다.
- 네이버/직방 링크는 원문 플랫폼 정책이나 매물 종료 여부에 따라 끊길 수 있습니다.
- 자동 수집 또는 화면 기반 정리 과정에서 누락이나 해석 오류가 있을 수 있습니다.
- 계약 안정성은 보고서만으로 판단할 수 없습니다.
- 보증보험 가능 여부는 HUG/HF/SGI 또는 중개사 문구가 아니라 실제 기관 심사 기준으로 확인해야 합니다.

## 같이 보는 사람을 위한 빠른 사용법

1. 배포 URL을 연다.
2. 먼저 `결론`과 `순위표`를 본다.
3. 관심 후보의 이름을 눌러 집별 분석으로 이동한다.
4. 장점보다 `리스크`와 `현장 확인` 항목을 먼저 본다.
5. 원문 링크에서 매물이 아직 살아 있는지 확인한다.
6. 방문 후보는 등기부, 선순위, 관리비 포함 항목, 방/욕실 수, 엘리베이터, 수압, 배수, 누수, 채광, 방범을 따로 체크한다.

## 운영 원칙

- 이 저장소는 공유 링크를 안정적으로 유지하는 것이 목적입니다.
- `index.html`은 생성된 산출물이므로, 작은 문구 수정 외에는 원본 생성 과정을 다시 거쳐 교체하는 것을 우선합니다.
- 원문 매물 링크는 접근성을 해치지 않는 선에서 유지합니다.
- 민감정보나 사적인 계약 자료는 이 저장소에 추가하지 않습니다.
